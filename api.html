Parameters

Parameters Description
informat required Determines in which format the input is given. Can be "text", "turtle", "rdfxml", "ntriples", "json", "html", "pdf" and much more.
"informat=text", "informat=turtle", "informat=rdfxml" must be implemented. If an unknown or not implemented "informat" is given, the following error has to be returned:
@prefix rlog: <http://persistence.uni-leipzig.org/nlp2rdf/ontologies/rlog#> . mylog:urn-xxx-yyy-zzz a rlog:Entry ; rlog:level rlog:ERROR ; rlog:date "2013-06-08T17:00:00Z"^^xsd:datetime ; rlog:message "Parameter 'informat=html' not implemented. " .
intype required Determines how the input is accessed or retrieved. Can be "direct", "url", "file". Webservices must implement "intype=direct" (e.g. via POST) and "intype=url", CLI must implement "intype=file" as well and access files with relative path names. It remains open how to implement 'direct' in a good way for CLI, maybe URL percent encoding as well? If an unknown or not implemented intype is given, the following error has to be returned:
mylog:urn-xxx-yyy-zzz a rlog:Entry ; rlog:level rlog:ERROR ; rlog:date "2013-06-08T17:00:00Z"^^xsd:datetime ; rlog:message "Parameter 'intype=whatever' not implemented. " .
input required Determines the input in accordance with the intype and informat parameter.
mylog:urn-xxx-yyy-zzz a rlog:Entry ; rlog:level rlog:ERROR ; rlog:date "2013-06-08T17:00:00Z"^^xsd:datetime ; rlog:message "Parameter 'input=This%20is%20a%20sentence.' is not a valid url, but 'intype=url' was given. " .
urischeme optional, case-sensitive, default: RFC5147String Determines the urischeme to be used. Must be the full URI of the scheme of the OWL class. In case the namespace is "http://persistence.uni-leipzig.org/nlp2rdf/ontologies/nif-core#" it can be omitted for brevity. Allowed values: "RFC5147String", "http://persistence.uni-leipzig.org/nlp2rdf/ontologies/nif-core#RFC5147String", etc.
mylog:urn-xxx-yyy-zzz a rlog:Entry ; rlog:level rlog:ERROR ; rlog:date "2013-06-08T17:00:00Z"^^xsd:datetime ; rlog:message "Parameter 'urischeme=RFC7415String' does not exist, did you mean RFC5147String?" .
prefix optional A prefix, which is used to create any URIs. Therefore the client should ensure, that the prefix is valid, when used at the start of the uri, e.g. http://test.de/test#. It is recommended that the client matches the prefix to the previously used prefix. If the parameter is missing it should be substituted by a sensible default (e.g. the web service uri).
urischemeparameters optional Some parameters for certain urischemes, e.g. contextlength
debug optional, default: false debug = "true" | "false" This options determines, if additional debug messages and errors should be written within the RDF.
